Image-based visual servoing for drone tracking.

03 / AUTONOMOUS VEHICLES

Will the pedestrian cross?

A trajectory-conditioned Transformer reads a pedestrian's past frames — position, heading, speed — and returns a distribution over next-second intent: cross, continue straight, or wait. The ego vehicle's planner receives the signal 1.5 s ahead of the decision point and brakes, yields, or proceeds accordingly.

  • TaskIntent · 3-way
  • Input30-frame traj
  • ModelTransformer
  • FrameworkPyTorch
  • BenchmarkJAAD / PIE
  • Horizon1.5 s ahead

●  QUADRUPED · CLAW
04 / LOCOMOTION

CLAW — 12-DOF quadruped dog robot.

Custom hardware, mechanical and electrical, built from scratch. Forward and inverse kinematics, jerk-optimized trajectories using quintic polynomials and Bezier curves, and a Cartesian PD controller for virtual joint compliance. Stereo-vision terrain costmap for footstep planning.

  • Quadruped
  • ROS
  • Kinematics
  • LQR
  • Stereo Vision
  • 3D Printing
●  FOOTSTEP · OCTOMAP
05 / LOCOMOTION

Doggo over uneven terrain.

Built a footstep affordance/costmap from a 3D occupancy octree. Slope features extracted via PCA over height deviations. Spline-based end-effector trajectories pick foothold positions that avoid obstacles. Tested on the Stanford Doggo with a Jetson TX2 and RealSense.

  • Stanford Doggo
  • OCTOMAP
  • PCA
  • Costmap
  • Trajectory Planner

07 / AUTONOMOUS

Self-driving e-scooter.

A scooter that repositions itself to riders or to designated parking spots. ENet semantic segmentation plus RGB-D builds a 2D sidewalk costmap. Dijkstra over OpenStreetMap data picks the global route. Custom kinematic model and hardware interface on Jetson Nano.

●  INDOOR NAV · A*
08 / AUTONOMOUS

Autonomous wheelchair for kids.

A ROS-based indoor navigation base built on a real differential-drive wheelchair. LiDAR + wheel encoder fusion for odometry, AMCL for localization, Gmapping for SLAM, RRT for exploration, A* for global planning. Written in C++14 with 95% test coverage and CI on Travis.

●  FRENET FRAME
09 / PLANNING

Jerk-minimized Frenet-frame trajectories.

Optimized local trajectories in the lane's TN frame for passenger comfort. Candidate paths get filtered by the car's kinematic constraints, then ranked for obstacle avoidance, lane changing, merging, and velocity keeping.

  • Trajectory Opt
  • Frenet
  • Self-Driving
  • Jerk Min
●  HYBRID A* · REEDS-SHEPP
10 / PLANNING

Hybrid A* with Reeds-Shepp.

A car planner combining holonomic and non-holonomic heuristics. Action space sampled from kinematic simulations and Reeds-Shepp paths. Holonomic costs avoid obstacles; non-holonomic costs grade trajectories on effort and human-preferred maneuvers.

  • A*
  • Reeds-Shepp
  • Kino-Dynamic
  • Self-Driving
●  POSE GRAPH · g2o
11 / SLAM

2D graph SLAM with ICP scan matching.

A SLAM pipeline using g2o pose-graph optimization fed by LiDAR and wheel odometry. ICP creates edges between poses with odometry-based initial guesses and gating thresholds. Loop closures snap drift back into alignment.

  • SLAM
  • g2o
  • ICP
  • LiDAR
  • Pose Graph
●  EKF · 15 STATES
12 / ESTIMATION

EKF tracking 15 states.

Pose, twist, and acceleration estimator using an omni-directional kinematic model for prediction, tuned for differential drive. Fused wheel odometry with IMU. Dynamic process-noise covariance prevents inflation when no absolute position arrives and the robot is stationary.

  • EKF
  • Bayesian
  • Sensor Fusion
  • IMU
●  MCL · LIKELIHOOD FIELD
13 / ESTIMATION

Particle filter localization.

Pose estimator built on omni-directional odometry with sampled Gaussian noise via the polar Box-Muller transform. The measurement model uses a LiDAR maximum-likelihood field with a pre-built lookup for fast scoring.

  • Particle Filter
  • MCL
  • LiDAR
  • Box-Muller
